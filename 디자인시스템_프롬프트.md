# 틱재팬 디자인 시스템 — Claude Agent 전달 프롬프트

> 사용법: 아래 코드블록 전체를 복사해 Claude Agent에 그대로 붙여넣으면 된다.
> 현재 운영 중인 틱재팬의 **실제 디자인 토큰**을 기반으로 작성되어, 기존 앱과 일관된 시스템이 나오도록 설계됨.

```
# 역할
너는 틱재팬(TickJapan)의 시니어 프로덕트 디자이너 겸 프론트엔드 엔지니어다.
지금 운영 중인 앱의 시각 언어를 정리해 "틱재팬 디자인 시스템"을 만든다.
새로 발명하지 말고, 현재 앱에 이미 존재하는 디자인을 추출·체계화·일관화하는 것이 목표다.

# 프로덕트 컨텍스트
- 틱재팬 = 한국어→일본어 번역 + 일본어 학습(피치 억양, 동사/형용사/명사/조사, 문법) 앱
- 사용자: 일본어를 독학하는 한국인. 모바일 우선(iOS/Android, Capacitor) + 반응형 웹
- 톤앤매너: 깔끔·신뢰감·친근함. 학습 도구답게 정보 가독성이 최우선. 화려함보다 명료함.
- 스택: React 18 + Vite. 현재는 인라인 스타일/일부 CSS 클래스 혼용. 컴포넌트는 /src/components.

# 현재 실제 사용 중인 토큰 (이걸 기준으로 체계화)
## 컬러
- Primary(브랜드): #5CA9CE  (가장 많이 쓰임 — 버튼/강조/링크/탭 활성)
- Primary 연한 배경 틴트: #F0F9FF, 반투명 틴트 #5CA9CE18 ~ #5CA9CE33
- Success/저장됨: #1D9E75 (보더), 배경 #E1F5EE, 텍스트 #085041
- Danger/에러: #C53030 ~ #C0306A 계열
- 텍스트: 기본 #111111, 보조 #555/#666, 약함 #888/#AAA, 힌트 #BBB
- 보더: 기본 #E8E8E8, 약함 #F0F0F0, 강조 #E0E0E0
- 서피스: 화이트 #FFFFFF, 연회색 #FAFAFA/#F8F9FA
## 타이포그래피
- 한국어: 'Noto Sans KR', 일본어: 'Noto Sans JP' (일본어 텍스트엔 반드시 JP 폰트)
- 굵기: 400/500/600/700 사용 중 (정리 필요)
## 형태
- 라운드: 카드 12~14px, 버튼 8~10px, 작은 요소 6px, pill 11~20px
- 그림자: 거의 안 씀(플랫). 모달/드로어에만 약한 그림자.
- 하단 시트(bottom sheet)·중앙 모달 패턴 사용(가입/ATT 안내 등)

# 해야 할 일 (산출물)
1. **디자인 토큰 정의** — 위 값을 의미 기반 토큰으로 재정의:
   - color (brand/primary, success, danger, text-{primary,secondary,tertiary,hint}, border-{default,subtle,strong}, surface-{base,raised,sunken}, tint)
   - typography scale (display/h1/h2/h3/body/caption — size·weight·line-height, 한/일 폰트 규칙)
   - spacing scale (4/8/12/16/20/24...)
   - radius scale (sm/md/lg/pill)
   - elevation(shadow) 1~2단계
   - motion (duration·easing 기본값)
   - 다크모드 대응 토큰도 함께 정의(현재 라이트 only지만 확장 대비)
2. **CSS 변수로 구현** — :root에 `--tj-*` 토큰을 정의한 `tokens.css`. 기존 인라인 스타일을 한 번에 안 바꿔도 점진 도입 가능하게.
3. **핵심 컴포넌트 스펙 + 예시 코드(React)** — 각각 토큰만 사용:
   - Button (primary/secondary/ghost/danger, sm/md/lg, loading/disabled)
   - Card (기본 카드, 패딩·보더·라운드 규칙)
   - Input / Textarea (번역 입력창 포함)
   - Tab / Chip (품사 필터·상단 탭처럼 wrap 되는 칩)
   - Badge / Pill (기본·응용 뱃지, 저장됨 상태)
   - List item (저장 목록 행)
   - Bottom Sheet & Modal (가입·ATT 안내에 쓰는 패턴)
   - Bookmark/Save 버튼 상태 (미저장 #5CA9CE / 저장됨 #1D9E75)
4. **사용 가이드라인** — do/don't, 컬러 사용 규칙(텍스트 대비), 한·일 폰트 적용 규칙, 간격 리듬, 접근성(터치 타깃 44px, 대비 AA, 폰트 최소 크기).
5. **마이그레이션 가이드** — 기존 인라인 스타일을 토큰으로 점진 교체하는 순서 제안(깨지지 않게).

# 제약·원칙
- 모바일 우선. 터치 타깃 ≥44px. 본문 폰트 ≥13px.
- 플랫 디자인(과한 그림자/그라데이션 금지). 정보 가독성 최우선.
- 일본어 텍스트엔 Noto Sans JP, 한국어엔 Noto Sans KR 일관 적용.
- 굵기는 400/500/700 3단계로 정리(600 남발 금지).
- 기존 운영 앱을 깨지 않게 — 토큰은 추가 도입 방식. 한 번에 전면 교체 강요 금지.
- 주석은 한국어로.
- 결과는 (a) tokens.css (b) 컴포넌트별 스펙+React 예시 (c) 가이드라인 문서(markdown) 형태로 제출.

# 먼저 할 것
구현 전에, 위 토큰을 바탕으로 한 **디자인 토큰 표(컬러/타이포/스페이싱/라운드)**와 **컴포넌트 목록**을 먼저 제안하고 내 확인을 받은 뒤 코드로 진행해라.
```
