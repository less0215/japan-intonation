## P3. 출시 후 최적화 — 앱 속도 등 (여유될 때)

### 속도 (체감 순서)
- [ ] **① 번역 응답 속도 — 서버 콜드스타트 개선** ⭐ 체감 가장 큼
  - Railway 무료/저가 플랜은 유휴 시 서버 슬립 → 첫 요청 느림
  - 해결: 플랜 상향(상시 가동) 또는 주기적 핑으로 깨워두기
- [ ] **② 첫 로딩 — JS 번들 코드 분할**
  - 현재 번들 1.25MB(gzip 430KB), 빌드 시 경고. lazy load로 첫 화면 단축
  - 단, Capacitor는 에셋 내장이라 우선순위 낮음

### 기능 강화 (선택)
- [ ] **저장 단어·예문을 계정 기반(서버)으로 전환**
  - 현재 localStorage(기기별) → 기기 바뀌면 안 따라감
  - 서버 저장으로 바꾸면 웹↔앱↔새폰 동기화. DB 스키마 추가 필요
- [ ] AppsFlyer 커스텀 이벤트 로깅 (광고 집행 시 가치 ↑)
- [ ] 휴대폰 SMS 인증 가입 (무분별 가입 추가 차단, SMS 비용·DB 변경 필요)
